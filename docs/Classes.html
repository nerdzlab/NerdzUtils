  • AtomicAsyncOperation

    A class incapsulate a logic of a single operation at a time If operation already in progress - next requests will just be added into completions list that will be called after operation finish This approach decreasing an omount of executions by not running same operation when it’s currently initiated by somebody else, and just notifying all requests completions after initial operation finish Useful, for example, when you need to lazy initialize some service when somebody requesting a data. In this case you can guarantee that service will not be initialised twice Or when you want to optimise requests executing. If you save all retrieved data to database - this class might be a good solution to avoid race conditions and unnecessary database storing execution NOTE: Operation is using backeground queue underhood, so wrap all your UI actiong into main queue
